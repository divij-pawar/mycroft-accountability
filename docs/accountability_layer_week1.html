Architecture constraints
The problem it solves

Without this — a naked conclusion

In a standard request-response cycle, if an agent flags a company's revenue as "unstable," the investor is forced to take its word for it. If the conclusion is based on a hallucinated footnote, the error is laundered through the system until it becomes an official — and incorrect — investment grade.

The mechanism

A signed affidavit — justification as a prerequisite

Every specialized agent must produce a ReasoningObject before its logic can propagate. If the agent cannot point to the specific line in a 10-K that justifies its conclusion, the orchestrator rejects the state transition.

C-02 · SDD v1.0
Known constraint — ADR-06

The thought_log is evidence of output, not process

The ReasoningObject enforces that a log exists, is structured, and is permanently recorded. What it cannot enforce is that the log is true. A hallucinated conclusion may be accompanied by a coherent, plausible-sounding thought_log. This limitation is encoded honestly in ADR-06 — not papered over.

ADR-06 — encoded Week 2

The audit layer makes the system legible. It does not make it correct. Reviewers must be trained on this distinction. This is not a gap to be filled in a later phase — it is a property of LLM-generated reasoning that no logging layer can fix.

The problem it solves

Post-hoc rationalization is a trap

When a stateless system fails, the standard debugging move is to ask the LLM why it made a certain choice. LLMs are world-class rationalizers — they will produce a plausible-sounding explanation for a mistake they already made, even if that explanation has nothing to do with the original reasoning.

The mechanism

Forensic state snapshots — time-travel auditing

Checkpointing snapshots the full state of the agentic committee at every decision node, preserving the raw context of the moment. Without it, you are investigating a crime scene after the rain has washed away the footprints.

Checkpoint saved at: STATE_NODE_3 run_id: "a3f2..." ticker: "AAPL" agents_active: [financial, patent, earnings] context_hash: "7b4f..." timestamp: "2026-04-26T09:14:33Z" → Human operator can rewind to exact step a logical contradiction entered the pipeline. State preserved as the machine saw it — not as it later remembered it.
C-03 RunRecord Store · append-only
Implementation decision

Append-only. No UPDATE. No DELETE.

The RunRecord store is PostgreSQL with a DB-layer trigger that unconditionally blocks UPDATE and DELETE on run_records. The application user has INSERT and SELECT permissions only. Immutability is enforced at the database layer, not the application layer.

SEC-03 · ADR-02
The problem it solves

Silent consensus hides the most dangerous signals

In a monolithic pipeline, conflicting data is averaged out. If the Patent Agent sees a crumbling IP moat but the Financial Agent sees high R&D spend, a standard model reports "steady innovation." This silent smoothing is where the most dangerous financial risks hide.

The mechanism

Hard-coded pause on semantic divergence

The Adversarial Arbitration Node triggers when two agents move in opposite semantic directions — delta between confidence scores and sentiment vectors exceeds threshold. Each agent receives the other's ReasoningObject and issues a challenge. Maximum iterations: 1.

AAN triggered: financial_agent: "revenue_trend: stable" (conf: 0.81) patent_agent: "ip_moat: declining" (conf: 0.76) Divergence delta exceeds threshold. → Each agent receives the other's ReasoningObject → Single debate loop: challenge directives issued → Outcome: Affidavit of Disagreement produced Result passed to Controller — not a halt. Investor sees: "AI agents reached conflicting conclusions on this analysis."
AAN · UN-03
Residual risk — T-04

Semantic poisoning is not eliminated

A structurally valid but semantically poisoned thought_log — produced when an agent ingests a fabricated source — will pass schema validation. The AAN reduces this risk by forcing cross-examination from different data streams, but does not eliminate it. If both agents ingest the same poisoned source, the AAN will not detect the contamination. Documented, accepted residual risk.

What this system is

Structural integrity — not fact verification

The Accountability Layer provides a stateful, append-only audit log of every Mycroft analysis run. It enforces structured reasoning via system directive injection and separates investor-facing from reviewer-facing rendering.

What this system is not

Four explicit constraints — encoded in SDD v1.0

These are not oversights. They are documented, accepted constraints from the first design session.

not a fact verification system not a hallucination prevention system not a grade override mechanism not a compliance archive (Phase 1)
Scope decision — Week 1

The system was initially named "Accountability Mesh." The scope was narrowed to "Accountability Layer" before Week 2 build began — reflecting an honest assessment of what could be built in four weeks. That correction is a design decision, not an error.
